# Growth Playbook 템플릿

> **목적**: MVP 런치 전후의 마케팅 실행 계획을 정의한다.
> 브랜드 정체성은 `brand_brief.md`에서, 기술 구현은 `prd.md`에서 참조한다.
> 이 문서는 "어떻게 알릴 것인가"에 집중한다.
>
> **사용법**: "brand_brief.md를 참조해서 이 템플릿으로 growth playbook을 만들어줘."
>
> **타이밍**: PRD 작성 후, 개발 Week 2 시작 전까지 완성한다. 런치 일정에 맞춰 역산한다.

## 1. 그로스 목표 & 지표

### 1.1 런치 목표 (런치 후 2주)

| 지표              | 목표치        | 측정 방법                          |
| ----------------- | ------------- | ---------------------------------- |
| 랜딩 방문자       | [예: 1,000명] | Vercel Analytics / GA              |
| 회원가입 전환율   | [예: 5%]      | 가입 수 / 방문자                   |
| 활성 사용자 (WAU) | [예: 50명]    | [핵심 액션]을 1회 이상 수행한 유저 |
| 유료 전환         | [예: 10명]    | 결제 완료 수                       |

### 1.2 핵심 액션 정의

> "활성 사용자"를 판단하는 기준 액션을 하나만 정한다.

| 항목           | 정의                                                                    |
| -------------- | ----------------------------------------------------------------------- |
| **핵심 액션**  | [예: 리포트 1개 생성 / 채널 1개 연결 / 첫 분석 확인]                    |
| **Aha Moment** | [예: 첫 리포트를 받아보고 자기 브랜드의 AI 검색 점유율을 확인하는 순간] |
| **활성 기준**  | [예: 주 1회 이상 핵심 액션 수행]                                        |

### 1.3 퍼널 정의

```
[인지] → [방문] → [가입] → [핵심 액션] → [유료 전환] → [리텐션]
  ↑         ↑        ↑          ↑              ↑            ↑
 채널     랜딩    온보딩     제품 가치      가격/플랜     습관 형성
```

| 퍼널 단계        | 예상 전환율  | 병목 예상          | 대응             |
| ---------------- | ------------ | ------------------ | ---------------- |
| 인지 → 방문      | [예: 3~5%]   | [채널 도달률]      | [채널 다각화]    |
| 방문 → 가입      | [예: 5~10%]  | [랜딩 카피 설득력] | [A/B 테스트]     |
| 가입 → 핵심 액션 | [예: 30~50%] | [온보딩 마찰]      | [온보딩 최적화]  |
| 핵심 액션 → 유료 | [예: 5~10%]  | [가치 체감 부족]   | [무료 제한 조정] |

## 2. 런치 채널 전략

> 솔로 개발자가 현실적으로 커버할 수 있는 채널만 선택한다. 3개 이하로 집중한다.

### 2.1 채널 우선순위

| 순위 | 채널                  | 타겟 매칭도 | 노력 대비 효과 | 역할                          |
| ---- | --------------------- | ----------- | -------------- | ----------------------------- |
| 1    | [예: Threads/X]       | [높/중/낮]  | [높/중/낮]     | [예: 빌더 브랜딩 + 런치 공지] |
| 2    | [예: 개발자 커뮤니티] |             |                | [예: 초기 트래픽 + 피드백]    |
| 3    | [예: 인스타그램]      |             |                | [예: 타겟 직접 도달]          |

### 2.2 채널별 전략

**채널 1: [예: Threads / X]**

| 항목        | 내용                                                       |
| ----------- | ---------------------------------------------------------- |
| 목적        | [예: 빌더 개인 브랜드 → 서비스 인지]                       |
| 빈도        | [예: 주 3~5회]                                             |
| 콘텐츠 유형 | [예: 빌딩 과정 공유, 인사이트, 런치 공지]                  |
| 톤          | → `brand_brief.md` 5장, 단 개인 계정이므로 좀 더 캐주얼 OK |
| KPI         | [예: 팔로워 증가율, 링크 클릭 수]                          |

**채널 2: [예: 커뮤니티 (Geeknews / Disquiet / Launchpit)]**

| 항목        | 내용                                            |

**채널 3: [예: 타겟 직접 도달 채널]**

| 항목 | 내용                                           |
| ---- | ---------------------------------------------- |
| 목적 | [예: Primary Persona에게 직접 도달]            |
| 방법 | [예: 인스타 DM / 이메일 아웃리치 / 오픈카톡방] |

## 4. SEO & 콘텐츠 전략

> 키워드 기반은 `brand_brief.md` 3장에서 가져온다. 여기서는 실행 계획만 정의한다.

### 4.1 테크니컬 SEO (런치 시 필수)

| 항목                     | 구현                                               | PRD 참조    |
| ------------------------ | -------------------------------------------------- | ----------- |
| 시맨틱 HTML              | [예: h1에 Primary 키워드 포함]                     | 랜딩 페이지 |
| meta title / description | → `brand_brief.md` 8.1 히어로 + 3.2 Primary 키워드 |             |
| OG 태그                  | → `brand_brief.md` 9.3                             |             |
| sitemap.xml              | 자동 생성 (Next.js)                                |             |
| robots.txt               | 기본 허용                                          |             |
| canonical URL            | 각 페이지에 설정                                   |             |
| structured data          | [예: SoftwareApplication / FAQPage]                |             |
| Core Web Vitals          | LCP < 2.5s, CLS < 0.1                              |             |
| 모바일 최적화            | 반응형 필수                                        |             |

### 4.2 콘텐츠 SEO (런치 전후 발행)

> MVP 단계에서는 최대 2~3편. 타겟이 검색할 키워드에 답하는 콘텐츠.

| 우선순위   | 콘텐츠                                    | 타겟 키워드 (→ 브리프 3.2) | 형태        | 발행 시점 |
| ---------- | ----------------------------------------- | -------------------------- | ----------- | --------- |
| 1          | [예: "[문제 키워드] 해결하는 N가지 방법"] | [예: AI 브랜드 모니터링]   | 블로그      | 런치 D-3  |
| 2          | [예: "[경쟁사] vs [우리] 비교"]           | [예: Otterly 대안]         | 비교 페이지 | 런치 D+3  |
| 3 _(선택)_ | [예: "[업계 트렌드] 가이드"]              | [예: GEO 최적화란]         | 가이드      | 런치 D+7  |

### 4.3 링크 빌딩 (자연 발생)

> 솔로 개발자가 할 수 있는 현실적인 백링크 전략만.

- 커뮤니티 포스트 (Geeknews, Disquiet, Launchpit) → 자연 백링크
- 개인 블로그/노션에 빌딩 로그 → 서비스 링크 포함
- Product Hunt 런칭 → 프로필 백링크
- GitHub에 관련 오픈소스 도구 → README에서 서비스 언급
