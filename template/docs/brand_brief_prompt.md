# 브랜드 브리프 작성 프롬프트

> **사용법**: [placeholder] 부분을 채운 뒤 AI에게 전달한다.
> `brand_brief_template.md`와 함께 사용한다.

---

## 프롬프트

```
아이템: [서비스 아이디어를 2~3문장으로 설명]

타겟: [누구를 위한 서비스인지. 구체적일수록 좋음]
- 예: "인스타그램에서 릴스로 제품을 파는 1인 셀러"
- 예: "AI 검색에서 자사 브랜드 노출이 궁금한 마케터"

경쟁/대안: [현재 이 문제를 해결하는 방법이나 경쟁 서비스가 있다면]
- 예: "수동으로 ChatGPT에 검색해보는 게 전부. 해외에는 Otterly, Peec AI 같은 서비스가 있음"
- 모르면 "잘 모름"이라고 적어도 됨

톤 방향: [원하는 느낌이 있다면. 없으면 비워도 됨]
- 예: "전문적이지만 딱딱하지 않은"
- 예: "개발자 커뮤니티에서 자연스러운 톤"

---

위 내용을 바탕으로 brand_brief_template.md 구조에 맞는 브랜드 브리프를 작성해줘.

작성 시 다음 원칙을 따라줘:

[네이밍]
- 후보를 최소 3개 제시하되, 각각의 장단점을 솔직하게 비교해줘.
- .com 또는 .ai 도메인 가용성, 한국어 발음 자연스러움, 검색 시 기존 서비스와 충돌 여부를 반드시 고려해줘.
- "좋은 이름"이 아니라 "왜 이 이름이 이 서비스에 맞는지"를 설명해줘.

[용어 사전]
- 핵심 개념별로 후보 단어를 먼저 나열하고, 하나를 확정하면서 나머지를 "사용하지 않는 표현"으로 명시해줘.
- 용어 선택의 기준을 밝혀줘. (예: "리포트"를 선택한 이유는 타겟이 익숙한 단어이기 때문)
- 확정 용어의 영문 코드명도 함께 정해줘. 이 코드명은 이후 DB 모델명, 라우트명, 컴포넌트명에 그대로 쓰인다.

[톤앤매너]
- 추상적인 형용사(예: "친근한")만 나열하지 말고, 반드시 실제 UI 상황별 O/X 예문을 작성해줘.
- 최소한 온보딩 인사, 에러 메시지, 빈 상태, 결제 유도, 성공 피드백 — 이 5가지 상황의 예문이 필요해.

[페르소나]
- "30대 직장인" 같은 인구통계가 아니라, 이 사람이 매일 겪는 구체적 상황과 불편함으로 묘사해줘.
- Anti-Persona(이 서비스의 타겟이 아닌 사람)도 반드시 정의해줘. 기능 범위가 불필요하게 넓어지는 걸 방지하기 위해서.

[포지셔닝]
- 경쟁사/대안 분석은 "우리가 더 좋다"가 아니라, 각각의 접근 방식 차이를 객관적으로 비교해줘.
- 포지셔닝 스테이트먼트는 For/Who/Our service is/That/Unlike/We 프레임으로 작성해줘.

[랜딩 카피]
- 히어로 헤드라인 후보를 2~3개 제시하고, 각각이 노리는 방향(호기심 유발 / 문제 직시 / 결과 제시)을 명시해줘.
- CTA 문구는 서비스 내부의 CTA와 동일하게 쓸 수 있는 문구로 통일해줘.

[전체 원칙]
- 모든 섹션은 "채워 넣기"가 아니라 "선택과 근거"로 작성해줘. 왜 이 단어를, 왜 이 톤을, 왜 이 포지셔닝을 선택했는지 이유가 있어야 해.
- 완성된 브리프는 이후 PRD 작성, 랜딩 페이지 개발, 마케팅 카피 작성 시 "참조 문서"로 쓰인다. 그 용도에 맞게 구체적이고 즉시 활용 가능한 수준으로 작성해줘.
```

## 사용 예시

```
아이템: AI 검색 엔진(ChatGPT, Perplexity, Gemini 등)에서 특정 브랜드가 어떻게 언급되는지 모니터링하고 주간 리포트를 제공하는 SaaS.

타겟: 자사 브랜드의 AI 검색 노출이 궁금한 스타트업 마케터, D2C 브랜드 대표

경쟁/대안: 수동으로 ChatGPT에 직접 검색해보는 게 전부. 해외에 Otterly, Peec AI가 있지만 한국 시장 특화 서비스는 없음.

톤 방향: 전문적이지만 권위적이지 않은. 데이터 기반이되 쉽게 읽히는.
```
