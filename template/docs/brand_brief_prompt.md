# 브랜드 브리프 생성 프롬프트

> **사용법**: `[입력]` 부분만 채워서 Claude에게 전달한다.

---

## 프롬프트

```
아이템: [서비스 아이디어를 2~3문장으로 설명]
타겟: [누구를 위한 건지 — 구체적일수록 좋음]
참고할 경쟁/레퍼런스: [있다면 서비스명이나 URL — 없으면 삭제]
시장: [한국 / 글로벌 / 특정 지역]

위 아이템으로 brand_brief.md 템플릿에 맞춰 브랜드 브리프를 작성해줘.

작성 원칙:
1. 네이밍은 후보 3개 이상을 내되, 각각 .com/.ai 도메인 가용성을 실제로 고려하고, 한국어로 발음했을 때 자연스러운지 평가해줘. 영어권에서도 통하면서 한국 시장에서 어색하지 않은 이름을 우선해.
2. 용어 사전은 이 서비스의 핵심 UX 흐름을 먼저 그려보고, 그 흐름에서 사용자가 만나는 모든 개념에 대해 확정 용어를 정의해줘. "사용하지 않는 표현" 칸에는 실제로 혼용될 가능성이 높은 유사어를 구체적으로 넣어줘.
3. 톤앤매너의 O/X 예시는 템플릿의 5개 상황(온보딩, 에러, 결제 유도, 빈 상태, 성공) 전부 이 서비스의 실제 맥락에 맞는 구체적인 문장으로 채워줘. 범용적인 예시가 아니라 이 서비스에서만 나올 수 있는 문장이어야 해.
4. 랜딩 카피는 히어로 헤드라인 후보를 3개 이상 제시하고, 각각 어떤 소구 방향(공포/호기심/이득/공감)인지 태그를 달아줘.
5. 페르소나는 추상적인 설명이 아니라, 이 사람의 하루 중 우리 서비스가 필요해지는 구체적인 순간(트리거 모먼트)을 서술해줘.
6. 경쟁 분석에서 포지셔닝 맵의 두 축은 이 시장에서 실제로 의미 있는 차별화 기준으로 설정해줘.
7. 검색 키워드 리서치(3장)에서는 타겟이 문제 인식 → 해결책 탐색 → 비교/결정 각 단계에서 실제로 검색할 법한 한국어/영어 키워드를 구분해서 넣어줘. 특히 "타겟이 쓰는 말 vs 업계 용어" 테이블은 실제 검색 행동을 기반으로 채우고, 3.5에서 이 키워드가 네이밍, 태그라인, 용어 사전, 랜딩 카피에 각각 어떻게 반영되어야 하는지를 구체적으로 연결해줘.
```

---

타겟: 브랜드 마케터, D2C 브랜드 운영자, SEO 담당자
참고할 경쟁/레퍼런스: Otterly.ai, Peec AI, Profound

## 프롬프트 설계 의도

각 작성 원칙이 어떤 문제를 해결하는지:

| 원칙                       | 없으면 생기는 문제                                                      | 있으면 달라지는 점                                                                       |
| -------------------------- | ----------------------------------------------------------------------- | ---------------------------------------------------------------------------------------- |
| 1. 네이밍 도메인+발음      | 멋진 이름인데 도메인 못 잡고, 한국어로 부르기 어색한 이름이 나옴        | 실제로 쓸 수 있는 이름만 후보에 올라옴                                                   |
| 2. UX 흐름 기반 용어 사전  | 기능 나열식으로 용어를 잡아서 실제 UI에서 빠지는 개념이 생김            | 사용자 여정의 모든 접점에서 용어가 커버됨                                                |
| 3. 서비스 특화 톤 예시     | "앗, 문제가 생겼어요" 같은 범용 문장만 나옴                             | "리포트 생성 중 문제가 생겼어요" 같은 맥락 있는 문장이 나옴                              |
| 4. 헤드라인 소구 방향 태그 | 비슷한 톤의 카피만 3개 나옴                                             | 공포/호기심/이득 등 다양한 각도의 카피를 비교 선택 가능                                  |
| 5. 트리거 모먼트           | "마케터, 30대, 디지털 마케팅" 같은 의미 없는 페르소나                   | "월요일 아침 주간 보고 준비하면서" 같은 실감나는 시나리오                                |
| 6. 의미 있는 포지셔닝 축   | "가격 ↔ 기능" 같은 뻔한 축이 나옴                                       | 이 시장에서 진짜 승부가 갈리는 기준으로 포지셔닝                                         |
| 7. 키워드 → 브랜딩 연결    | 키워드 리서치를 해놓고 네이밍/카피에 반영 안 함. SEO와 브랜딩이 따로 놈 | 타겟이 실제로 검색하는 언어가 서비스명, 태그라인, UI 용어, 랜딩 카피에 자연스럽게 녹아듦 |

## 선택적 추가 지시

상황에 따라 프롬프트 끝에 추가할 수 있는 옵션:

**경쟁사 웹 리서치가 필요할 때**

```
작성 전에 [경쟁사 URL]을 직접 확인하고, 그들의 실제 포지셔닝/가격/기능을 기반으로 경쟁 분석을 작성해줘.
```

**키워드 볼륨을 실제로 조사할 때**

```
3.2 키워드 맵 작성 시 웹 검색을 통해 실제 검색 볼륨과 경쟁 난이도를 조사해줘. 추정이 아니라 데이터 기반으로 채워줘.
```

**기존 브리프를 업데이트할 때**

```
기존 brand_brief.md를 읽고, [변경할 부분]만 수정해줘. 나머지는 유지.
```

**여러 아이템을 비교 검토할 때**

```
위 아이템으로 브랜드 브리프의 1장(서비스 핵심 정의)과 7장(경쟁 포지셔닝)만 먼저 작성해줘.
전체 브리프는 아이템 확정 후 진행할게.
```

**브리프 완성 후 바로 다음 단계로 이어갈 때**

```
브리프 완성 후, 이 brand_brief.md를 참조해서 growth_playbook.md 템플릿으로 growth playbook도 이어서 작성해줘.
```

**Notion에 저장할 때**

```
작성 완료 후 Notion의 [페이지명/URL]에 저장해줘.
```

## 전체 워크플로우 요약

```
Step 1: 이 프롬프트로 brand_brief.md 생성
        ↓
Step 2: brand_brief.md 참조하여 PRD 생성 (2week-mvp-prd-template.md)
        ↓
Step 3: brand_brief.md + PRD 참조하여 growth_playbook.md 생성
        ↓
Step 4: 개발 시작 (PRD 따라 구현, brand_brief 용어/톤 적용)
        ↓
Step 5: 런치 (growth_playbook 시퀀스 실행)
```

각 문서의 역할:

- **brand_brief.md** → 우리가 누구인가 (정체성 + 언어 + 키워드)
- **prd.md** → 무엇을 만드는가 (기술 스펙 + 기술 SEO)
- **growth_playbook.md** → 어떻게 알리는가 (채널 + 시퀀스 + 그로스)
