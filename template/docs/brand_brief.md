# 브랜드 브리프 템플릿

> **목적**: 개발 시작 전에 서비스의 정체성을 확정한다.
> 이 문서에서 정의한 네이밍, 용어, 톤은 이후 PRD, 랜딩 페이지, 마케팅 카피, UI 텍스트 전체의 기준이 된다.

---

## 1. 서비스 핵심 정의

### 1.1 문제 정의

**누구의, 어떤 문제인가?**

[타겟이 현재 겪고 있는 구체적인 상황과 고충을 서술한다. 추상적인 문제가 아니라 "이 사람이 매일/매주 반복하는 비효율" 수준으로 구체화한다.]

**지금은 어떻게 해결하고 있는가?**

[엑셀, 수동 작업, 경쟁 서비스, 무시하고 넘어감 등. 기존 해결 방식의 한계를 명시한다.]

**우리가 해결하는 방식은?**

[위 문제를 어떤 접근으로 풀 것인지. 기술이 아니라 "사용자 입장에서의 경험 변화"로 서술한다.]

### 1.2 엘리베이터 피치

> [누구]를 위한 [무엇]입니다.
> [기존 방식]과 달리, [차별화 포인트]로 [핵심 가치]를 제공합니다.

### 1.3 핵심 가치 제안 (Value Proposition)

한 문장으로: [이 서비스를 쓰면 [타겟]은 [구체적 결과]를 얻는다.]

## 4. 톤 & 보이스

### 4.1 브랜드 성격

[서비스를 사람에 비유하면 어떤 사람인지 3~4개의 키워드로 정의한다.]

예: "똑똑한 동료" — 전문적이지만 권위적이지 않고, 필요한 걸 빠르게 알려주는 사람.

| 키워드         | 의미                      | 이런 느낌 (O)          | 이런 느낌 아님 (X)                       |
| -------------- | ------------------------- | ---------------------- | ---------------------------------------- |
| [예: 명쾌한]   | [복잡한 걸 쉽게 정리해줌] | "3가지로 요약했어요"   | "다양한 관점에서 종합적으로 분석하면..." |
| [예: 친근한]   | [거리감 없이 편한]        | "확인해볼까요?"        | "확인하시기 바랍니다"                    |

| 항목          | 기준                                                        |

### 4.3 톤 적용 예시

**온보딩 첫 인사**

- O: [예: "반가워요! 먼저 채널 하나를 연결해볼까요?"]
- X: [예: "서비스를 이용해주셔서 감사합니다. 아래 절차에 따라 채널을 등록해주세요."]

**에러 발생 시**

- O: [예: "앗, 문제가 생겼어요. 다시 시도해주세요."]
- X: [예: "오류가 발생했습니다. 관리자에게 문의하세요."]

**결제 유도**

- O: [예: "무료 플랜으로 충분하지 않다면, Pro에서 더 많은 걸 해보세요."]
- X: [예: "상위 플랜을 구독하시면 추가 기능을 이용하실 수 있습니다."]

**빈 상태 (데이터 없음)**

- O: [예: "아직 데이터가 없어요. 첫 리포트를 만들어볼까요?"]
- X: [예: "데이터가 존재하지 않습니다."]

**성공/완료**

- O: [예: "저장했어요 ✓"]
- X: [예: "성공적으로 저장되었습니다."]

## 5. UI 용어 사전 (Key Terms Glossary)

> 서비스 전체에서 하나의 개념에는 하나의 단어만 쓴다.
> 이 테이블은 PRD, 코드, 랜딩 페이지, 마케팅 카피 전체의 기준이 된다.

### 5.1 핵심 용어

| 개념            | 확정 용어      | 사용하지 않는 표현               | 영문 (코드용) | 사용 맥락            |
| --------------- | -------------- | -------------------------------- | ------------- | -------------------- |
| [핵심 개념 A]   | [예: 리포트]   | 분석결과, 인사이트, 보고서, 리뷰 | `report`      | 대시보드, 알림, 랜딩 |
| [핵심 개념 B]   | [예: 채널]     | 소스, 플랫폼, 매체               | `channel`     | 연결 설정, 리포트    |
| [핵심 개념 C]   | [예: 모니터링] | 추적, 감시, 트래킹, 워칭         | `monitoring`  | 기능 소개, CTA       |
| [사용자 액션 1] | [예: 연결하기] | 추가, 등록, 설정, 셋업           | `connect`     | 버튼, 온보딩         |
| [사용자 액션 2] | [예: 내보내기] | 다운로드, 추출, 익스포트         | `export`      | 리포트 상세          |
| [과금 단위]     | [예: 크레딧]   | 포인트, 토큰, 횟수, 건           | `credit`      | 가격, 사용량         |

### 5.2 플랜/티어 명칭

| 티어            | 이름          | 한 줄 설명                | 타겟                |
| --------------- | ------------- | ------------------------- | ------------------- |
| Free            | [예: Starter] | [예: 기본 기능 체험]      | [가벼운 사용자]     |
| Paid 1          | [예: Pro]     | [예: 본격적인 모니터링]   | [Primary Persona]   |
| Paid 2 _(선택)_ | [예: Team]    | [예: 팀 협업 + 고급 분석] | [Secondary Persona] |

### 5.3 네비게이션 라벨

| 위치       | 라벨                  | 비고                  |
