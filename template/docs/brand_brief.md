# 브랜드 브리프 템플릿

> **목적**: 개발 시작 전에 서비스의 정체성을 확정한다.
> 이 문서에서 정의한 네이밍, 용어, 톤은 이후 PRD, 랜딩 페이지, 마케팅 카피, UI 텍스트 전체의 기준이 된다.
>
> **사용법**: "아이템: [설명]. 이 템플릿으로 브랜드 브리프를 만들어줘."

---

## 1. 서비스 핵심 정의

### 1.1 문제 정의

**누구의, 어떤 문제인가?**

[타겟이 현재 겪고 있는 구체적인 상황과 고충을 서술한다. 추상적인 문제가 아니라 "이 사람이 매일/매주 반복하는 비효율" 수준으로 구체화한다.]

**지금은 어떻게 해결하고 있는가?**

[엑셀, 수동 작업, 경쟁 서비스, 무시하고 넘어감 등. 기존 해결 방식의 한계를 명시한다.]

**우리가 해결하는 방식은?**

[위 문제를 어떤 접근으로 풀 것인지. 기술이 아니라 "사용자 입장에서의 경험 변화"로 서술한다.]

### 1.2 엘리베이터 피치

> [누구]를 위한 [무엇]입니다.
> [기존 방식]과 달리, [차별화 포인트]로 [핵심 가치]를 제공합니다.

### 1.3 핵심 가치 제안 (Value Proposition)

한 문장으로: [이 서비스를 쓰면 [타겟]은 [구체적 결과]를 얻는다.]

### Primary Persona

| 항목                   | 내용                                              |
| ---------------------- | ------------------------------------------------- |
| 이름 / 역할            | [예: 김민수 / 1인 이커머스 셀러]                  |
| 나이대 / 맥락          | [예: 30대, 스마트스토어 월 매출 500만원대]        |

| 서비스를 발견하는 경로 | [예: 인스타 릴스, 블로그 검색, 커뮤니티 추천]     |

| 항목              | 내용                                  |
| ----------------- | ------------------------------------- |

| Primary와 다른 점 | [니즈나 사용 패턴의 차이]             |

## 3. 검색 키워드 & 타겟 언어 리서치

> 타겟이 이 문제를 해결하려 할 때 실제로 검색하는 단어를 파악한다.
> 여기서 발견한 키워드와 표현이 이후 네이밍(4장), 용어 사전(6장), 랜딩 카피(8장)에 직접 반영된다.

### 3.1 타겟의 검색 행동

| 항목                                 | 내용                                                                  |
| ------------------------------------ | --------------------------------------------------------------------- |
| **문제 인식 단계에서 검색하는 말**   | [예: "인스타 스팸 댓글 왜 이렇게 많아", "AI 검색 브랜드 노출"]        |
| **해결책 탐색 단계에서 검색하는 말** | [예: "인스타 댓글 관리 도구", "AI SEO 모니터링 툴"]                   |
| **비교/결정 단계에서 검색하는 말**   | [예: "Otterly vs Peec 비교", "[경쟁사명] 대안", "[경쟁사명] pricing"] |

### 3.2 핵심 키워드 맵

| 키워드                    | 검색 볼륨 추정 | 의도 (정보/비교/구매) | 난이도     | 우리 서비스와의 관련도 |
| ------------------------- | -------------- | --------------------- | ---------- | ---------------------- |
| [예: AI 브랜드 모니터링]  | [높/중/낮]     | [정보]                | [높/중/낮] | [핵심 / 관련 / 롱테일] |
| [예: AI 검색 점유율]      |                |                       |            |                        |
| [예: ChatGPT 브랜드 언급] |                |                       |            |                        |
| [예: GEO 최적화]          |                |                       |            |                        |
| [예: AI SEO 도구]         |                |                       |            |                        |

### 3.3 타겟이 쓰는 언어 vs 업계 용어

> 타겟이 실제로 쓰는 말과 업계/전문가가 쓰는 말이 다를 수 있다. 랜딩 카피에는 타겟 언어를, 블로그/SEO 콘텐츠에는 둘 다 쓴다.

| 개념                            | 타겟이 쓰는 말                   | 업계/전문가 용어             | 우리 서비스에서 쓸 표현              |
| ------------------------------- | -------------------------------- | ---------------------------- | ------------------------------------ |
| [예: AI가 브랜드를 언급하는 것] | "AI가 우리 브랜드를 추천해줌"    | "LLM citation", "AI mention" | [랜딩: 타겟 언어 / 앱 내: 확정 용어] |
| [예: 검색 결과에서의 위치]      | "AI 검색에서 몇 번째로 나오는지" | "AI search ranking", "GEO"   |                                      |
|                                 |                                  |                              |                                      |

### 3.4 SEO 콘텐츠 방향 (MVP 런치 시점)

> MVP 단계에서 무거운 콘텐츠 전략은 불필요하다. 런치와 동시에 최소한으로 깔아둘 것만 정의한다.

**랜딩 페이지 SEO**

- Primary 키워드: [랜딩 히어로/메타에 반영할 핵심 1~2개]
- Secondary 키워드: [기능 소개 섹션에 자연스럽게 녹일 것]
- 메타 디스크립션 방향: [타겟 언어 기반, 검색 의도에 답하는 형태]

**런치 시 발행할 콘텐츠 (선택)**

- [예: "[문제 키워드] 해결하는 법" — 블로그 1편]
- [예: "[경쟁사] vs [우리] 비교" — 비교 페이지 1편]

### 3.5 네이밍/카피에 대한 시사점

> 위 리서치 결과가 이후 섹션에 어떻게 반영되어야 하는지 정리한다.

- **네이밍(4장)에 반영**: [예: "모니터링"이라는 단어가 검색 볼륨이 높으므로 서비스명/태그라인에 포함 고려]
- **태그라인(4장)에 반영**: [예: 타겟이 "AI 검색"이라는 표현을 주로 쓰므로 "AI 검색 브랜드 모니터링"이 "GEO 최적화"보다 직관적]
- **용어 사전(6장)에 반영**: [예: 앱 내에서 "GEO"보다 "AI 검색 점유율"이라는 표현이 타겟에게 더 와닿음]
- **랜딩 카피(8장)에 반영**: [예: 히어로 헤드라인에 "AI 검색" + "브랜드" 조합 사용, Pain 섹션에 문제 인식 단계 검색어 활용]

### 5.1 브랜드 성격

### 5.3 톤 적용 예시

**에러 발생 시**

- O: [예: "앗, 문제가 생겼어요. 다시 시도해주세요."]

**결제 유도**

- O: [예: "무료 플랜으로 충분하지 않다면, Pro에서 더 많은 걸 해보세요."]

**빈 상태 (데이터 없음)**

- O: [예: "아직 데이터가 없어요. 첫 리포트를 만들어볼까요?"]

**성공/완료**

- O: [예: "저장했어요 ✓"]
- X: [예: "성공적으로 저장되었습니다."]

## 6. UI 용어 사전 (Key Terms Glossary)

> 서비스 전체에서 하나의 개념에는 하나의 단어만 쓴다.
> 이 테이블은 PRD, 코드, 랜딩 페이지, 마케팅 카피 전체의 기준이 된다.
> 3.3에서 파악한 "타겟이 쓰는 말"을 기반으로, 앱 내 용어를 확정한다.

### 6.1 핵심 용어

| 개념            | 확정 용어      | 사용하지 않는 표현               | 영문 (코드용) | 사용 맥락            |
| --------------- | -------------- | -------------------------------- | ------------- | -------------------- |
| [핵심 개념 A]   | [예: 리포트]   | 분석결과, 인사이트, 보고서, 리뷰 | `report`      | 대시보드, 알림, 랜딩 |

## 8. 랜딩 페이지 카피 가이드

> 위에서 정의한 톤, 용어, 포지셔닝, 그리고 3장의 키워드를 그대로 적용한다.

### 8.1 히어로 섹션

- **헤드라인**: [예: "AI가 당신의 브랜드를 어떻게 말하는지, 알고 계세요?"]
  - → 3.2 핵심 키워드 중 [키워드]를 반영
- **서브카피**: [예: "ChatGPT, Perplexity, Gemini — AI 검색에서 브랜드 노출을 실시간으로 모니터링하세요."]
  - → 3.1 타겟의 검색 표현을 활용
- **CTA**: [예: "무료로 시작하기"] — 6.3 네비게이션 CTA와 동일한 문구

### 8.2 Pain → Solution
